General information: a subject (in spanish asignatura o materia) is failed with a grade (in spanish calificacion o nota) less than 70

The name of the tables in the database are always in singular

These are the tables of the database, with its structure:

Table name: usuario 
columns of the usuario table:
    id: primary key, autogenerated
    nombres: related to user name
    apellidos: related to user surname
    correo: related to user email

relationships of the usuario table (this columns dont belong directly to usuario table, they are in external tables referenced in the description of every column):
    roles: many-to-many relationship with the roles table through usuario_roles table
    grupos: many-to-many relationship with the grupos table through usuario_grupos table
    asignaturas: many-to-many relationship with the asignaturas table through usuario_asignaturas table

Table name: calificacion
columns of the calificacion table:
    id: primary key, autogenerated
    idDocente: foreign key, many-to-one relationship with the usuario table, references a docente (profesor)
    idEstudiante: foreign key, many-to-one relationship with the usuario table, references a estudiante (estudiante)
    idAsignatura: foreign key, many-to-one relationship with the asignatura table
    year: year of the grade or evaluation
    idPeriodo: foreign key, many-to-one relationship with the periodo table, indicates the period of the evaluation
    calificacion: numeric grade or score for the student in the given subject and period

Table name: periodo
columns of the periodo table:
    id: primary key, autogenerated
    periodo: name or description of the period, such as a semestre or periodo (in this table we have Periodo 1, Periodo 2, Periodo 3, Periodo 4 for periodo values)

Table name: asignatura
columns of the asignatura table:
    id: primary key, autogenerated
    asignatura: name or description of the subject, such as "Matematicas" or "historia"

Table name: grupo
columns of the grupo table:
    id: primary key, autogenerated
    grupo: name or identifier of the group, such as "Group A" or "Grado 10" or "Grado 8A" but for our database and this table grupo we have Grupo A Grupo B and Grupo C

Table name: role
columns of the role table:
    id: primary key, autogenerated
    role: name or identifier of the role, such as "admin", "profesor", or "estudiante"

Table name: usuario_roles
columns of the usuario_role table:
    usuarioId: integer, many-to-many relationship with usuario table
    roleId: integer, many-to-many relationship with role table

Table name: usuario_grupos
columns of the usuario_grupos table:
    usuarioId: integer, many-to-many relationship with usuario table
    grupoId: integer, many-to-many relationship with grupo table 

Table name: usuario_asignaturas
columns of the usuario_asignaturas table:
    usuarioId: integer, many-to-many relationship with usuario table
    asignaturaId: integer, many-to-many relationship with asignatura table
